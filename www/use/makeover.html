Extreme Makeover of wsmonitor

The wsmonitor workspace is undergoing "Extreme Makeover". You can try the stable release or bleeding edge builds. The reason/highlights of this makeover are:
  • Maven: Using Maven instead of Ant for building the workspace and generating the distribution
  • Modular architecture: Making wsmonitor extensible by using HK2. This will allow other users to extend the functionality by providing additional modules.
  • Ease-of-use: A platform-independent jar allows to run the tool using "java -jar" command instead of platform-dependent scripts.
  • JDK Logging: JDK logging is used to print verbose message instead of "-verbose" switch.
